# CrabRes · 从 Claude Code 512K 行源码中学到的系统设计

> 逐一映射 Claude Code 的每个关键特性到 CrabRes 的增长场景

---

## 1. KAIROS → CrabRes Daemon（永远在线的增长引擎）

**Claude Code 原版**：KAIROS 是一个后台 daemon，每 5 分钟 tick 一次，订阅 GitHub webhooks，在用户不在时自主工作，在用户在时协作。

**CrabRes 版本：Growth Daemon**

```
┌─────────────────────────────────────────────┐
│           CrabRes Growth Daemon              │
│                                             │
│  心跳：每 30 分钟 tick 一次                  │
│                                             │
│  tick 时做什么：                              │
│  ├── 扫描竞品网站是否有变化（定价/功能/新页面）│
│  ├── 搜索 Reddit/X 上新出现的相关讨论         │
│  ├── 检查用户的关键词排名变化                 │
│  ├── 检查内容日历是否有到期任务               │
│  └── 如果发现值得行动的事 → 通知用户          │
│                                             │
│  用户不在时（未聚焦）：                       │
│  · 自主执行低风险任务（更新竞品档案、          │
│    准备下一批社媒草稿、更新 SEO 数据）         │
│                                             │
│  用户在时（聚焦）：                           │
│  · 展示发现，等待确认后再行动                 │
│                                             │
│  午夜边界：                                   │
│  · 生成今日增长日报                           │
│  · 刷新明天的任务清单                         │
└─────────────────────────────────────────────┘
```

## 2. AutoDream → Growth Dream（夜间记忆蒸馏）

**Claude Code 原版**：当用户空闲时，agent 进行"记忆巩固"——合并分散的观察、消除矛盾、将模糊洞察转化为结构化事实。四阶段：orient → gather → consolidate → prune。

**CrabRes 版本：Growth Dream**

```
触发条件：24 小时未做整理 + 5 次以上会话

四阶段：

Phase 1: Orient（定向）
  扫描记忆目录，读取当前增长计划、竞品档案、用户画像

Phase 2: Gather（采集）
  从最近会话中提取：
  · 用户的新决策（"我不想做付费广告了"）
  · 策略效果反馈（"Reddit 帖子效果很好"）
  · 新发现的竞品信息
  · 用户否决的建议

Phase 3: Consolidate（整合）
  · 将"上周"这种相对时间转为绝对时间
  · 删除已过时的竞品数据（价格已变）
  · 合并重复的用户画像信息
  · 更新增长计划状态（哪些完成了、哪些放弃了）

Phase 4: Prune（修剪）
  · 控制记忆文件总量在合理范围
  · 优先保留：有效策略 > 用户偏好 > 竞品数据 > 过期信息
```

**为什么重要**：不做这个，20-30 次会话后记忆会充满矛盾和过时信息，CrabRes 会给出前后不一致的建议。

## 3. ULTRAPLAN → Growth Strategy Session（深度策略会）

**Claude Code 原版**：将复杂规划任务卸载到云端 Opus 4.6，给 30 分钟深度思考时间，本地轮询结果。

**CrabRes 版本：Deep Strategy**

```
触发词："deep strategy" 或 "重新想想整个增长策略"

场景：
  用户说："产品方向变了，从 B2C 转 B2B，需要重新制定增长计划"
  
  这不是 30 秒能回答的问题。CrabRes 需要：
  · 重新研究 B2B 竞品格局
  · 重新定义目标用户
  · 重新评估所有渠道的 ROI
  · 重新设计飞轮
  · 重新生成全部执行物料

  → 触发 Deep Strategy 模式
  → 后台运行 10-30 分钟
  → 完成后通知用户查看新计划
  → 用户可以边干别的事边等

好处：用户不用盯着 loading 等 30 分钟
```

## 4. Buddy System → Growth Mascot（增长吉祥物）

**Claude Code 原版**：基于用户 ID 确定性生成的 Tamagotchi 宠物，有稀有度、属性、ASCII 动画、性格。

**CrabRes 版本：Growth Buddy**

```
不是单纯的宠物。CrabRes 的 Buddy 是你的增长伙伴的"人格化身"。

核心设计：
├── 基于用户产品类型确定性生成
│   · SaaS 产品 → 火箭型 Buddy
│   · 电商产品 → 购物车型 Buddy  
│   · 社区产品 → 对话气泡型 Buddy
│
├── 属性反映你的增长状态（不是随机的，是实时的）
│   · GROWTH：周增长率
│   · REACH：本周触达的用户数
│   · CONSISTENCY：连续执行增长任务的天数
│   · INSIGHT：CrabRes 对你产品理解的深度
│   · MOMENTUM：增长飞轮的运转状态
│
├── 进化机制
│   · 连续 7 天执行增长任务 → Buddy 升级
│   · 达到 100 用户里程碑 → Buddy 获得新外观
│   · 连续 3 天没登录 → Buddy 显示担心的表情
│
├── 可分享（病毒传播）
│   · "我的 Growth Buddy 达到 Lv.10 了！"
│   · 自带 CrabRes 品牌水印
│   · 一键分享到 X/LinkedIn
│
└── 情绪化互动
    · 你发布了一个高互动帖子 → Buddy 欢呼
    · 竞品降价 → Buddy 拉响警报
    · 长时间没活动 → Buddy 发通知 "想我了吗？你的竞品可没休息"
```

**为什么重要**：提高用户粘性 + 提供病毒传播素材 + 把枯燥的增长任务游戏化。

## 5. Coordinator Mode → Growth Roundtable（增长圆桌）

**Claude Code 原版**：用 prompt（而非代码）驱动的多 Agent 编排。关键规则：
- "不要橡皮图章糟糕的工作"
- "必须理解发现后才能指派后续工作"
- "永远不要说'基于你的发现'——必须具体到文件路径、行号"

**CrabRes 版本**：

```
圆桌规则（写在 Coordinator prompt 里）：

1. "永远不要给用户泛泛的建议。
    '做 Reddit 营销'不是建议。
    '在 r/cscareerquestions 每周发 2 篇帖子，格式为经验分享，
     避免在标题中提及产品名'才是建议。"

2. "在推荐任何渠道之前，必须先用研究工具验证：
    · 竞品是否在这个渠道有存在感？
    · 目标用户是否真的活跃在这里？
    · 用户的预算是否够在这个渠道投入？
    如果不能用数据回答这三个问题，不要推荐。"

3. "不要假设用户的产品有市场。
    研究阶段必须主动寻找'这个方向可能不行'的证据。
    找到后必须诚实告知用户。"

4. "每个策略必须附带具体的 KPI 和时间线。
    不接受'长期来看会有效果'这种说法。
    必须说'第 1-2 周预期 X，第 3-4 周预期 Y'。"

5. "所有文案输出必须是可直接复制粘贴使用的。
    不要占位符，不要 [你的产品名]，
    使用用户产品的真实信息。"
```

## 6. Team Memory Sync → Growth Context Sync（增长上下文同步）

**Claude Code 原版**：多 Agent 之间的共享草稿本（scratchpad）+ 团队记忆同步 + 结构化消息传递。

**CrabRes 版本**：

```
所有专家共享的 Growth Context：
┌─────────────────────────────────────┐
│  growth-context/                    │
│  ├── product.md      产品 DNA       │
│  ├── competitors/    竞品档案       │
│  │   ├── comp-a.md                  │
│  │   └── comp-b.md                  │
│  ├── audience.md     用户画像       │
│  ├── channels.md     渠道数据       │
│  ├── budget.md       预算分配       │
│  ├── strategy.md     当前策略       │
│  ├── results.md      执行效果       │
│  └── scratchpad.md   专家讨论区     │
└─────────────────────────────────────┘

规则：
· 任何专家都可以读取所有文件
· 专家通过 scratchpad 讨论（用户可选看）
· 只有 Coordinator 可以更新 strategy.md
· 任何专家发现竞品变化可以直接更新 competitors/
· results.md 由用户反馈和数据追踪自动更新
```

## 7. Frustration Detection → Growth Mood Sensing（增长情绪感知）

**Claude Code 原版**：用正则检测用户是否在骂它。

**CrabRes 版本——更深层的情绪理解**：

```
检测维度：

1. 焦虑检测
   信号："为什么还没有用户" / "竞品已经融资了" / "是不是方向不对"
   响应：切换到安抚+数据驱动模式，展示进展数据（哪怕很小）

2. 失去动力检测
   信号：7 天未登录 / 连续跳过每日任务 / 回复变短
   响应：Buddy 发温和的推送 + 展示竞品的最新动作激发竞争意识

3. 方向迷茫检测
   信号："要不要换方向" / "不确定" / 频繁切换策略
   响应：暂停当前计划，触发 Deep Strategy 做系统性重新评估

4. 过度乐观检测
   信号：用户说 "下个月目标 10 万用户" 但现在 50 个都没有
   响应：温和地校准预期，给出基于数据的合理目标

5. 执行疲劳检测
   信号：任务完成率下降 / "太多了做不过来"
   响应：自动精简计划，从 5 件事砍到最重要的 1-2 件
```

## 8. Write-Ahead Logging → Growth Journal（增长日志）

**Claude Code 原版**：每次 API 调用前把对话状态写入磁盘，进程被杀也能恢复。

**CrabRes 版本**：

```
growth-journal/
├── 2026/
│   └── 04/
│       ├── 2026-04-01.md   ← 每日追加日志
│       ├── 2026-04-02.md
│       └── ...
│
每条日志记录：
  · 时间
  · 事件类型（研究发现/策略变更/用户反馈/指标变化）
  · 详情
  · 影响

作用：
  1. Growth Dream 整理记忆时的数据源
  2. 用户可以回顾"我的增长历程"
  3. 生成月度/季度增长复盘报告
  4. 作为可分享的"增长故事"（病毒传播素材）
```

## 9. Permission Modes → Growth Trust Levels（增长信任分级）

**Claude Code 原版**：5 级权限从"每步确认"到"完全信任"。YOLO Classifier 用 ML 判断是否需要确认。

**CrabRes 版本**：

```
信任等级随使用时间逐步升级：

Level 1: 新用户（第 1 周）
  · 每个策略建议需要确认
  · 每篇文案需要审核后才能标记"就绪"
  · 不会主动做任何事

Level 2: 建立信任（第 2-4 周）
  · 研究和分析自动进行
  · 文案自动生成但需要审核
  · 竞品变化自动追踪并通知

Level 3: 深度合作（第 2 月+）
  · 自动生成下周全部社媒内容
  · 自动更新增长计划（基于数据）
  · 发现机会自动准备行动方案

Level 4: 自主增长（用户主动开启）
  · 自动在 Reddit 相关帖子下回复（自然风格）
  · 自动提交到 AI 目录
  · 自动更新 SEO 页面
  · 用户只需每周审核一次
```

## 10. Prompt Cache Economics → Context Budget for Growth（增长上下文预算）

**Claude Code 原版**：Prompt cache 是 P0 级别的关注点，追踪 14 种缓存击穿向量，用 "sticky latches" 延迟模式切换以保护缓存。

**CrabRes 版本**：

```
增长研究的 Token 预算管理：

总预算（每次完整策略制定）：~100K tokens

分配：
  竞品研究：30K（最大头，需要抓取和分析多个竞品）
  用户研究：20K（社媒搜索+分析）
  策略制定：15K
  文案生成：25K（所有帖子/邮件/文章）
  审核验证：10K

优化策略：
  · 竞品数据缓存：分析过的竞品不重复分析（除非检测到变化）
  · 用户画像缓存：同品类产品共享用户研究
  · 文案模板缓存：成功的文案格式复用
  · 增量更新：策略迭代只分析变化部分，不从头开始
```

## 11. 六大 Agent 软件模式 → CrabRes 的实践

这是从整个 Claude Code 分析中提炼出的六个定义性模式：

| 模式 | Claude Code | CrabRes |
|------|------------|---------|
| **简单循环 + 复杂工具** | while(true) 只有几百行，50万行都在工具和控制上 | 增长循环逻辑简单，复杂性在研究工具、渠道 API 接入、文案生成 |
| **上下文是有限资源** | 直接注入 + 文件检索 + 摘要 + 驱逐 | 产品 DNA 始终注入，竞品数据按需加载，历史策略只保留结论 |
| **Prompt 即架构** | 编排算法是 prompt 不是代码 | 专家角色/圆桌规则/质量标准全部写在 prompt 里 |
| **确定性包裹非确定性** | 每次模型调用都包裹在验证/重试/限制中 | 每个策略建议都要经过数据验证和逻辑检查 |
| **审核是必须的** | AI 写代码，人类审核 | AI 写增长策略和文案，用户审核并执行 |
| **防御自己的工具** | 7 层安全防线 | 防止 Agent 乱花钱/乱发帖/给出错误建议 |
