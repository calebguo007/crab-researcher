# CrabRes — AI Growth Strategy Agent

## 项目概述

**项目名称**: CrabRes（小蟹研究员）
**定位**: AI 增长策略 Agent，帮独立开发者/初创公司做市场研究 + 制定增长计划 + 执行并追踪效果
**核心差异化**: 不是给建议的聊天机器人，是**做研究 + 写内容 + 追踪效果**的 Agent。13 个专家组成圆桌会议，各自有性格和专业视角，会争论、互相挑战，最终综合出可执行策略。

**代码路径**: `/Users/guoyi/Desktop/market/crab-researcher/`
**技术栈**: FastAPI + React + Tailwind + OpenRouter/Moonshot(LLM) + Tavily(搜索) + Neon PostgreSQL + Vercel(前端) + Render(后端)

### 核心问题（2026-04-04 实测发现）

**Agent 不做研究，只会问问题。** 真实用户对话中，用户说"accio是竞品"，Agent 没有去搜索 accio、没有分析产品，反而重复问"你在做什么"。根本原因：

1. **Tavily API Key 未配置** — 搜索工具不可用，Agent 无法获取外部信息
2. **工具调用链路未跑通** — 从未有过一次真实的"搜索→分析→输出"完整流程
3. **缺少执行器** — `write_post` / `write_email` 只生成文案不执行，Agent 无法与真实世界交互
4. **无数据反馈闭环** — 无法追踪发布内容的效果，Growth Daemon 的行动追踪是空的

**一句话：大脑很强，手脚没接上。架构 85 分，实际体验 20 分。**

## 执行优先级（2026-04-04 更新）

### P0：让 Agent 能"看到"和"做到"（从聊天机器人变成真 Agent）

当前最大的问题不是专家不够多、UI 不够好看，而是 **Agent 无法与真实世界交互**。

1. **配通 Tavily API Key + 验证搜索→分析→输出的完整链路**
   - 目标：用户说"accio是竞品"，Agent 立刻搜索 accio，返回产品分析
   - 这是 10 分钟能完成的事，但决定了产品能不能用

2. **接通 X/Twitter API（读+写）**
   - 读：获取用户帖子数据、粉丝增长、竞品动态
   - 写：`write_post` 从"生成文案"升级为"确认后真正发布"
   - Trust Level 落地：Level 1 建议 → Level 2 一键确认 → Level 3 自主发布

3. **Playwright 升级为"截图+多模态理解"**
   - 当前 `browse_website` 只抓 HTML，大多数现代网站拿不到有效内容
   - 改为：打开页面 → 截图 → 多模态 LLM 理解内容，覆盖没有 API 的平台

4. **Growth Daemon 闭环**
   - 发布内容 → 24h 后拉取数据 → judge_result 打分 → 更新记忆 → 下次建议调整
   - 这是 Agent 和工具的根本区别

### P1：对话体验（让 Agent 不像机器人）

5. **去掉模板化回复** — Agent 不应该说"您好，根据您提供的产品信息"，应该直接做事
6. **拿到信息就行动** — 用户说了产品名就开始搜索，不要反复确认
7. **专家对话可见** — 用户能看到专家各自分析、争论的过程

### P2：数据接入

8. **Google Analytics / Search Console** — 真实流量数据
9. **Stripe / LemonSqueezy** — 真实收入数据
10. **MCP 生态** — 让用户自己接 Notion/Slack/GitHub 等工具

### P3：视觉打磨

- 替换 emoji → 真实图标
- 替换 SVG 生物体 → 精致插画
- 暗色模式完善
