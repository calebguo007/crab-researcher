# CrabRes 调研总结 + 优化计划

> 基于 50+ 次搜索、20+ 个产品/文章深度分析

---

## 一、竞品格局发现

### 直接竞品（AI 增长/营销工具）

| 产品 | 做什么 | 定价 | CrabRes 的优势 | CrabRes 该学的 |
|------|--------|------|---------------|---------------|
| **Averi.ai** | 初创内容营销引擎（SEO+GEO+发布） | 免费 beta | Averi 只做内容，CrabRes 做全渠道 | 品牌记忆、CMS 直接发布、GEO 优化 |
| **Jasper** | 企业品牌内容平台 | $59/月/人 | 太贵、偏企业。CrabRes 面向 indie | 品牌声音训练系统 |
| **Copy.ai** | GTM 销售+营销工作流 | $36/月/5人 | 偏销售不偏增长策略 | 工作流自动化模板 |
| **ReplyAgent** | Reddit 自动回复营销 | 按成功帖子付费 | 只做 Reddit，我们做全渠道 | **托管账户系统、Google 排名帖子分析、反封号机制** |
| **OptaReach** | 多渠道自动外联（LI/Reddit/X/Email） | 未公开 | 只做外联，不做策略 | **ICP 评分、统一收件箱、AI 回复代理** |
| **Reddbot** | Reddit 全自动评论 | 未公开 | 太激进（容易封号） | 24/7 监控+自动检测 |
| **VenturOS** | Vibe Coder 内容生成 | 未公开 | 只生成内容不做研究 | 连接 GitHub 仓库的创意 |
| **Accio Work** | 阿里国际企业级 AI Agent | 免费 beta | 面向跨境电商不是通用增长 | **本地优先、浏览器自动化、多 Agent 群聊、Cron 任务** |

### 关键洞察

1. **没有人在做「验证方向+策略+执行物料」全链路** — 这是 CrabRes 的真空区
2. **Reddit 自动化是热门赛道** — ReplyAgent/OptaReach 专门做这个，说明需求真实
3. **GEO（生成引擎优化）是新趋势** — Averi 把它作为核心卖点，我们也应该有
4. **品牌记忆是刚需** — 每个竞品都在解决"每次重新解释"的问题

## 二、2026 SaaS 设计发现

### 转化率最高的设计模式（来自 PxlPeak 12 个最佳案例）

| 模式 | 效果 | CrabRes 该怎么做 |
|------|------|-----------------|
| **首屏交互式 Demo** | 试用注册率 3.2x（vs 静态截图） | Landing 首屏放一个可交互的 Agent 对话 Demo |
| **对话式首页** | 转化率 4.1x（vs 功能列表） | 首屏就是 Prompt 输入框 |
| **AI 引导入职** | 激活率 +42% | 入职时 AI 问 3 个问题个性化体验 |
| **社会证明瀑布流** | 每屏都有信任信号 | 证言放在定价表上方（不是底部） |
| **对比页面** | 转化率 2-3x（vs 首页） | 建立 "CrabRes vs ChatGPT" 对比页 |
| **ROI 计算器** | 页面停留时间 +52% | "CrabRes 能帮你省多少时间/钱"计算器 |

### 最该立刻改的

**1. Landing Page 首屏应该是可交互的 Demo，不是文字**
- 目前是标题+描述+按钮（每个 SaaS 都这样）
- 应该是：一个输入框，用户输入产品描述，立刻看到 Agent 开始搜索
- 这会让转化率提升 3-4 倍

**2. 需要暗色模式作为默认（开发者偏好暗色）**
- Cursor、Linear、Vercel 都是暗色默认
- 我们的目标用户是 indie hacker/developer
- 暗色 + glassmorphism = 高级感

## 三、Harness Engineering 发现

### 六大工程支柱（最新总结）

1. **上下文架构** — CLAUDE.md 宪法 + 动态注入 + 压缩 ← 我们有但粗糙
2. **架构约束** — 工具白名单 + Schema 限制 + 权限层 ← 我们有基础
3. **自验证循环** — Critic Agent + 对抗性验证 ← 我们有 Critic 但没做验证循环
4. **上下文隔离** — 子 Agent 独立上下文，不互相污染 ← 未实现
5. **记忆管理** — AutoDream + MEMORY.md + 选择性加载 ← 有骨架但不完善
6. **可观测性** — 每个工具调用都记录日志 ← 部分实现

### CrabRes 最该改进的

- **自验证循环**：Agent 输出策略后，自动让 Critic 审核，不通过则重来
- **上下文隔离**：每个专家应该只看到和自己相关的上下文，不是全部
- **选择性记忆加载**：不是每次加载所有记忆，只加载相关的

## 四、Claude Code 还没学到位的

| 特性 | 我们当前状态 | 应该做到 |
|------|------------|---------|
| **流式工具执行** | 串行（等完一个再下一个） | 并发执行多个工具（asyncio.gather） |
| **ULTRAPLAN 深度规划** | 没有 | 复杂策略调整→后台 30 分钟深度思考→通知结果 |
| **Hooks 生命周期** | 没有 | 工具调用前/后可插入自定义逻辑 |
| **Prompt Cache** | 没有 | 相同产品信息不重复发送给 LLM |
| **15 秒阻塞预算** | 没有 | 主动通知超过 15 秒自动后台化 |
| **Speculative Execution** | 没有 | 用户还在打字时就开始预测性搜索 |

## 五、优化优先级

### P0：立刻做（影响最大）

1. **Landing 首屏改为交互式 Demo** — 放一个输入框，输入产品名立刻看到搜索过程
2. **并发工具执行** — web_search + social_search 同时跑，缩短 Agent 响应时间
3. **Critic 自动验证** — 每次输出策略前自动让 Critic 审核

### P1：本周做

4. **暗色模式作为默认**（面向开发者群体）
5. **对比页面** — "CrabRes vs ChatGPT" 独立页面
6. **GEO 优化能力** — 内容策略专家增加 GEO 知识

### P2：下周做

7. **Reddit 深度整合** — 学 ReplyAgent 的高排名帖子分析
8. **ICP 评分** — 学 OptaReach 的用户画像评分
9. **品牌记忆增强** — 学 Averi 的品牌声音系统

### P3：之后做

10. **ULTRAPLAN** — 复杂策略后台深度思考
