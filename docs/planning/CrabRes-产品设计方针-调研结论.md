# CrabRes 产品设计方针（2026-04-03 调研结论）

> 如果你是乔布斯，你会怎么设计？如果你是 Anthropic 的首席产品官，你会怎么设计？

---

## 核心原则

### 1. Progressive Disclosure（渐进式披露）
**来源**：Apple HIG、Nielsen Norman Group、Agentic Design Patterns

不要一上来就展示所有能力。用户不需要知道"这里有 13 个专家"——他们需要看到**正在发生什么**。

**CrabRes 的应用**：
- 圆桌可视化：不是静态装饰品。当 Agent 正在调度专家时自动出现，没有专家工作时不可见
- 专家输出：默认折叠为 2 行摘要，用户感兴趣才展开
- 功能层级：核心路径（产品→研究→策略）简单明了，高级功能（@私聊、Deep Strategy）等用户需要时自然发现

**三层结构**：
```
Layer 1（首次看到）：对话 + 引导卡片
Layer 2（Agent 工作时）：专家面板自动出现 + 研究状态
Layer 3（深度使用后）：@ 私聊 + Roundtable 可视化 + Playbook 管理
```

### 2. Claude 式对话风格
**来源**：Anthropic Claude.ai 交互风格分析

- 简短回答简短问题。"hi" → "Hi! What are you building?" 完毕
- 长篇回答复杂问题。给了完整产品信息+明确目标 → 完整策略
- 永远不 dump information。用户没问的不答
- 温暖但不油腻。专业但不冷淡

### 3. 暖色设计系统（Tavily 启发）
**来源**：Tavily API Platform 设计分析

| 元素 | 之前（冷蓝白） | 现在（暖米色） |
|------|---------------|---------------|
| 背景 | #FAFBFE（蓝灰白） | #FEFCF8（奶油米色） |
| 品牌色 | #0EA5E9（天蓝） | #D97706（琥珀橙） |
| 字体 | Space Grotesk + DM Sans | Inter（Claude 同系） |
| 阴影 | 单层薄影 | 双层柔影（更有深度） |
| 纹理 | 网格+噪点 | 无（干净） |
| 卡片 | 玻璃态半透明 | 暖白半透明 |

### 4. 字体 = 品牌人格
**来源**：Claude 使用定制 Söhne 字体，Tavily 使用 Suisse Int

Inter 的选择理由：
- 和 Claude 的 Söhne 属于同一审美家族（几何无衬线但有温度）
- 开源免费
- 400 字重正文不冷硬（DM Sans 在大段文字里太方正了）
- 600 字重标题有力量感但不压迫

### 5. 首次体验 = 生死线
**来源**：UX Design Institute Onboarding Best Practices 2025

用户打开产品的前 30 秒决定去留。关键设计：

1. **不要空白输入框**——给出 3 个示例（"I'm building an AI resume optimizer..."），点击即填入
2. **不要说教**——不要解释"CrabRes 有 13 个专家、圆桌会议..."，让他们用着用着自己发现
3. **不要信息过载**——首次对话只需要知道：说出你的产品 → 我帮你研究
4. **10 秒内给价值感知**——用户说了产品名 → 立即看到"正在搜索竞品..."的状态 → 知道有事在发生

## 设计反模式（我们之前犯的错）

| 反模式 | 为什么差 | 怎么改 |
|--------|---------|--------|
| 静态圆桌挂在左边 | 用户不知道这是什么，看起来像装饰 | Progressive Disclosure：有活动才出现 |
| 说 hi 就输出 500 字 | 信息过载，用户有压力 | Claude 风格：匹配回复长度 |
| 蓝白配色 | 廉价感，和 ChatGPT 没区别 | 暖米色 + 琥珀橙，有辨识度 |
| 网格纹理背景 | "我是 AI 工具" 的标配外壳 | 干净背景，让内容说话 |
| 13 个专家名字全列在侧栏 | 看不懂，不知道干嘛 | 只在他们工作时才出现 |
