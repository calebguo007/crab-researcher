# 前置规划

## 1. 基础部署（Docker Compose）

```bash
# 进入项目
cd /Users/guoyi/Desktop/market/crab-researcher

# 启动核心服务（PostgreSQL + Redis + API + Frontend）
docker compose up -d

# 仅重建后端（改代码后常用）
docker compose up -d --build api
```

- 调度方式：**应用内 APScheduler**（无需 n8n）
- API 文档：`http://localhost:8002/docs`
- 前端：`http://localhost:3000`

## 2. LLM API 准备（Demo阶段）

- **Moonshot API Key**（已接入，当前主力）
- OpenAI / DeepSeek（可选后备）

## 3. 消息平台 Webhook

- **飞书群机器人（先做）**：创建群 → 添加自定义机器人 → 获取 Webhook URL
- 把 URL 填入 `.env` 的 `FEISHU_WEBHOOK_URL`
- 然后执行：

```bash
docker compose up -d --build api
```

## 4. 数据库策略

- 当前：Docker 内 PostgreSQL + pgvector（本地开发/演示）
- 后续：可迁移到云 PostgreSQL 或 Supabase（非当前必需）

## 5. RAG / 混合检索预埋

- 已规划并接入接口：`POST /api/rag/search`
- 支持模式：`vector` / `keyword` / `hybrid`
- 向量不可用时自动降级关键词检索，保证可用性

## 6. 前端重设计阶段建议

- 先不改后端契约，直接复用新联调接口：
  - `GET /api/system/dashboard/alerts`
  - `GET /api/system/dashboard/price-trends`
  - `GET /api/system/scheduler/status`

## 7. 域名（可选，后期）

- 买域名用于 Dashboard 和 Webhook 回调
- 生产期再配 Nginx + HTTPS
