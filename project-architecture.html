🔄 核心调用流 · 4 大监测链路

💰 价格监测
  1. 1
    Scheduler 每分钟扫描到期任务 next_run_at ≤ now
  2. 2
    Scraper 三层策略: 直链抓取 → 搜索页解析 → LLM分析
  3. 3
    BeautifulSoup 解析HTML提取商品名+价格
  4. 4
    解析失败 → LLM 从页面文本提取 Tier2
  5. 5
    _detect_price_change() 对比历史 >10% 触发告警
  6. 6
    Notification → 飞书价格变动卡片（含链接）
  7. 7
    结果写入 monitoring_results 表
📢 舆情监测
  1. 1
    Scheduler 或手动 POST /monitoring/sentiment
  2. 2
    Scraper 抓取微博/小红书搜索页
  3. 3
    提取帖子文本 → 传入 LLM 做情感分析
  4. 4
    返回: 情感评分(0-1) + 热门话题 + 提及量
  5. 5
    评分 <0.4 或提及 >300 → 触发 sentiment_spike
  6. 6
    Notification → 飞书舆情预警卡片
🆕 新品监测
  1. 1
    Scheduler 或手动 POST /monitoring/new_product
  2. 2
    Scraper 抓取当前搜索结果商品列表
  3. 3
    _detect_new_products() 对比最近5次历史数据
  4. 4
    不在历史中的商品 → 标记为新品
  5. 5
    Notification → 飞书新品发现卡片
📝 报告生成
  1. 1
    用户调用 POST /reports/generate
  2. 2
    ReportGenerator 收集监测数据 + 竞品数据
  3. 3
    RAG 混合检索知识库，增强上下文
  4. 4
    LLM 生成 Markdown 报告 Tier3/4
  5. 5
    CostController 记录 Token 消耗 + 费用
  6. 6
    存入 reports 表，返回完整报告
